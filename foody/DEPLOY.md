# Foody 上线指南

Foody 能被「谁」访问，取决于你把它放在哪里跑。从易到难分三层，按你的阶段选一个就行。

| 层次 | 谁能访问 | 费用 | 适合 |
|------|----------|------|------|
| ① 同一个 WiFi | 你 + 同 WiFi 的人 | 免费 | 自己测、家里人测 |
| ② 临时公网 | 任何人（你电脑开着时） | 免费 | 给朋友试用、收集反馈 |
| ③ 正式常驻 | 任何人，24 小时 | 约 US$7/月 | beta 反馈好之后，真正上线 |

---

## ① 同一个 WiFi（你现在就在用）

双击 `start-foody.bat`，手机连同一个 WiFi，打开电脑显示的 `http://192.168.x.x:3000` 即可。
**限制**：只有同一个 WiFi 的设备能开；电脑关机就断。

---

## ② 临时公网 —— 免费，今天就能让全世界访问

让朋友用手机数据（不在你家 WiFi）也能打开，但**不用部署、不花钱**。原理：在你电脑和互联网之间开一条安全隧道，自动给你一个 `https://` 网址。

**一次性安装**（只需做一次）：打开 PowerShell，运行
```
winget install --id Cloudflare.cloudflared
```

**每次要分享时**：
1. 双击 `start-foody.bat`（服务器跑起来）
2. 双击 `share-online.bat`
3. 窗口里会出现一个 `https://xxxx.trycloudflare.com` —— 把它发给朋友，谁都能打开

**限制**：你电脑 + 这两个窗口开着才有效；网址每次会变。**适合 beta 找朋友试用**。

---

## ③ 正式常驻 —— 24 小时在线（推荐 Render）

当你 beta 试得满意、想要一个一直在线、网址固定的正式站，就部署到云端。我已经把配置文件（`render.yaml`）准备好了，跟着做：

### 第 1 步：把代码推到 GitHub
1. 注册 https://github.com （免费）
2. 新建一个仓库（Repository），名字随意，例如 `foody`，**设为 Private（私有）**
3. 按 GitHub 页面给的命令把本地代码推上去（我可以帮你一步步弄）

### 第 2 步：在 Render 上部署
1. 注册 https://render.com ，用 GitHub 账号登录
2. 点 **New → Blueprint**，选你刚推的 `foody` 仓库
3. Render 会自动读取 `render.yaml`，确认后点 **Apply**
4. 等几分钟，它会给你一个固定网址，例如 `https://foody.onrender.com`
   - 自带 HTTPS（绿锁），全马来西亚、任何网络都能访问
   - 数据存在挂载的持久磁盘上，重启 / 更新都不丢

### 关于费用（诚实说明）
- `render.yaml` 里用的是 **starter 实例（约 US$7/月）**，因为**持久磁盘需要付费实例**。
- 免费实例（free）虽然 0 元，但：闲置会休眠（朋友打开要等约 30 秒冷启动），而且**没有持久磁盘——每次更新代码数据会被清空**。所以免费只适合临时试，不适合存真实用户数据。
- 想换更便宜 / 别的平台（Fly.io 等）也可以，告诉我即可。

### 之后想换成自己的域名（可选）
买一个域名（如 `foody.my`，约 RM50/年），在 Render 的 Custom Domain 里绑定即可。这步等有需要再说。

## 我已经帮你加好的「上线必备」安全措施

这些都在代码里做好了，部署时自动生效：

- **自动备份**：每 6 小时把数据库复制一份到 `data/backups/`，保留最近 24 份。万一文件损坏可恢复。
- **HTTPS 安全 Cookie**：生产模式下登录凭证只在加密连接传输（`NODE_ENV=production` 时自动开启）。
- **防刷限流**：注册（每 IP 每小时 10 次）、发帖（每 10 分钟 30 次）、留言（每 10 分钟 60 次）、登录错误（10 分钟 20 次）。
- **安全响应头**：CSP、防点击劫持、禁嗅探等，挡掉常见网页攻击。
- **健康检查**：`/api/health`，部署平台用它判断服务有没有挂。
- **隐私**：电话号码只在登录后、帖子里才出现；搜索结果绝不泄露号码。

## 数据 & 备份在哪里
- 数据库：`foody/data/db.json`（云端则在持久磁盘）
- 上传的照片 / 视频：`foody/uploads/`
- 自动备份：`foody/data/backups/`
- 想手动备份：把整个 `data/` 和 `uploads/` 文件夹复制走即可。
